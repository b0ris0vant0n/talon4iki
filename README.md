# Telegram-бот для проверки талонов на gorzdrav.spb.ru

Скрипт сначала обращается к API портала записи по направлению и проверяет наличие талонов без открытия браузера. Если API недоступно или ответ изменился, он переключается на резервный браузерный сценарий.

Основная проверка смотрит:

- есть ли кнопка `Выбрать талон`;
- или сайт показывает ошибку `Отсутствуют свободные талоны`.

Если талон появился, бот отправляет уведомление в Telegram и запоминает состояние, чтобы не слать одно и то же сообщение каждые 10 минут. В чате у бота также есть кнопка для ручной проверки.

## Быстрый старт

1. Создай виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

2. Установи зависимости:

```bash
pip install -r requirements.txt
```

3. Создай `.env` на основе шаблона:

```bash
cp .env.example .env
```

Шаблон `.env.example` специально обезличен. Реальные номер направления, фамилию и токены храни только в `.env` или `.env.server` и не отправляй их в GitHub.

4. Заполни в `.env`:

- `REFERRAL_NUMBER`
- `LAST_NAME`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID` — если не знаешь, оставь пустым, сначала напиши любое сообщение своему боту, потом скрипт сам попробует взять `chat_id` из последних обновлений.
- `TELEGRAM_PROXY` — необязательно. Если сервер не может достучаться до Telegram напрямую, укажи HTTP/HTTPS-прокси, например `http://login:password@host:port`.
- `NOTIFY_ON_EVERY_CHECK` — если поставить `true`, бот будет присылать результат каждой автоматической проверки, даже когда талонов нет.

5. Разовая проверка:

```bash
python bot.py --once
```

6. Постоянный режим:

```bash
python bot.py
```

По умолчанию проверка идет раз в 10 минут. Это регулируется переменной `CHECK_INTERVAL_MINUTES`.

После запуска бот пришлет меню с кнопкой `Проверить талоны`. Ее можно нажать в любой момент, чтобы выполнить внеочередную проверку вручную. Также работают команды `/start`, `/menu` и `/check`.

Если бот находит свободные талоны, он дополнительно спрашивает `Записаться?` кнопками `Да/Нет`. По кнопке `Да` он отправляет ссылку на страницу записи и данные направления для быстрого копирования.

## Запуск на сервере через Docker

1. Скопируй проект на сервер.

2. Создай серверный env-файл:

```bash
cp .env.example .env.server
```

3. Заполни в `.env.server`:

- `REFERRAL_NUMBER`
- `LAST_NAME`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `TELEGRAM_PROXY` — необязательно, только если нужен прокси для Telegram API
- `CHECK_INTERVAL_MINUTES`
- `NOTIFY_ON_EVERY_CHECK` — необязательно, включи `true`, если хочешь получать сообщение после каждой автоматической проверки

`.env.server` не должен попадать в репозиторий: это серверный файл с реальными секретами.

4. Запусти контейнер:

```bash
docker compose --env-file .env.server up -d --build
```

5. Проверить логи:

```bash
docker compose logs -f
```

Контейнер настроен с `restart: unless-stopped`, поэтому бот будет подниматься автоматически после перезагрузки сервера.

Перед обычным деплоем можно прогнать тесты в Docker:

```bash
docker compose --profile test run --rm tests
```

Или использовать готовый скрипт:

```bash
chmod +x deploy.sh
./deploy.sh
```

Скрипт сначала запускает тесты, и только если они проходят, пересобирает и поднимает контейнер бота.

## Как работает уведомление

- Когда талонов нет, скрипт только обновляет `state.json`.
- Когда талон появляется впервые после периода отсутствия, скрипт шлет сообщение в Telegram.
- Если включить `NOTIFY_ON_EVERY_CHECK=true`, бот будет присылать статус после каждой автоматической проверки.
- Пока статус не меняется, повторных уведомлений не будет.

## Ограничения

- Сайт может менять верстку, тексты кнопок и поведение формы.
- Иногда портал уходит на регламентные работы. Для такого случая скрипт тоже выводит отдельный статус.
- Если на сайте появится CAPTCHA или дополнительная защита, понадобится доработка.
- Браузерный резервный сценарий не входит в Docker-образ по умолчанию. На сервере основной путь проверки идет через API.
